# 后端服务层说明

## 三层架构

```
routes → controllers → services → repositories → MySQL
```

| 层级 | 职责 | 示例 |
|------|------|------|
| **routes/** | 路由注册 + 权限中间件绑定 | `templateRouter.get("/templates", requirePermission("template:view"), ...)` |
| **controllers/** | 请求参数提取、校验、响应格式化 | `templateController.listTemplates(req, res)` |
| **services/** | 业务逻辑、编排多个repository | `templateService.listTemplates(page, pageSize, keyword)` |
| **repositories/** | 原生SQL查询、数据库交互 | `templateRepository.findAll({ page, pageSize, keyword })` |

## 关键服务

### schemaService — 运行时Schema编译

`services/schemaService.js` — 核心service，负责：
- 编译模块元数据：从 `print_business_module` + `print_field_dict` 读取
- 生成前端表单配置：根据字段类型/必填/可搜索属性动态生成
- 生成搜索配置：仅 `searchable=1` 的字段出现在搜索区
- 编译数据校验规则：用于新增/编辑业务数据时的后端校验

新增自定义业务模块后，无需修改任何后端代码，schemaService 自动支持新模块。

### recordService — 统一业务数据CRUD

`services/recordService.js` — 所有业务类型的增删改查统一入口：
- `list(bizType, query)` — 支持 keyword 批量查询 + fieldFilters 精确查询
- `create(bizType, fields)` — 校验必填 + 唯一性（is_unique字段）
- `update(bizType, bizCode, fields)` — 按编码更新
- `remove(bizType, bizCode)` — 单条删除
- `batchRemove(bizType, bizCodes)` — 批量删除

### pdfGenerator + pdfLayout — PDF生成

- `services/pdfLayout.js` — 解析 `print_template_element`，计算元素坐标、样式、字体
- `services/pdfGenerator.js` — 调用 PDFKit 生成PDF，集成 bwip-js（条码）+ qrcode（二维码）

### authService — 认证鉴权

- `login(username, password)` — 验证密码 → 生成JWT → 查询权限 → 返回token+permissions+menus
- `verifyToken(token)` — JWT验证
- `getPermissionCodesForUser(userId)` — 遍历用户角色获取所有权限码

## 中间件

| 文件 | 函数 | 说明 |
|------|------|------|
| `auth.js` | `requireAuth` | JWT验证，注入 `req.user` 和 `req.permissions` |
| `auth.js` | `requirePermission(code)` | 权限验证（支持单个或数组，函数动态判断） |
| `asyncHandler.js` | `asyncHandler` | 捕获异步错误统一抛给errorHandler |
| `errorHandler.js` | `errorHandler` | 统一错误响应格式 |

## 工具函数

| 文件 | 导出 | 说明 |
|------|------|------|
| `utils/response.js` | `sendSuccess(res, data)` | 统一成功响应 `{ code:0, message:"success", data }` |
| `utils/response.js` | `appError(message, code, status)` | 生成业务错误 |
| `utils/dsl.js` | DSL模板元素校验函数 | 验证模板元素的字段绑定、位置、尺寸等 |

## 配置文件

| 文件 | 说明 |
|------|------|
| `config/env.js` | 读取 `.env` 环境变量，提供类型安全的配置导出 |
| `config/db.js` | MySQL 连接池创建（mysql2/promise） |
| `config/testDb.js` | 外部测试数据库连接（仅测试环境使用） |
