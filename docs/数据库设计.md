# 数据库设计

本文档说明 打印模板中心的数据库结构。

## 数据库信息

```text
Database: wms_print_template
```

迁移脚本目录：

```text
backend/migrations/
```

迁移文件：`001_init.sql`（核心 schema）+ `002_rbac_tables.sql`（权限系统）。

## business_record 统一表

所有业务数据（库位/容器/商品/自定义模块）统一存储于此表：

```sql
CREATE TABLE business_record (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  module_code VARCHAR(32) NOT NULL,        -- LOCATION/CONTAINER/PRODUCT/自定义
  record_code VARCHAR(128) NOT NULL,        -- 业务主键编码值
  record_data JSON NOT NULL,               -- 全部字段键值对
  search_text TEXT NULL,                   -- 搜索文本（可搜索字段值拼接）
  status TINYINT NOT NULL DEFAULT 1,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_module_record (module_code, record_code),
  INDEX idx_module_code (module_code)
);
```

## print_business_module

```sql
CREATE TABLE print_business_module (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  module_code VARCHAR(32) NOT NULL,
  module_name VARCHAR(128) NOT NULL,
  template_label VARCHAR(128) NOT NULL,
  data_label VARCHAR(128) NOT NULL,
  record_code_field VARCHAR(64) NOT NULL,  -- 主键字段编码
  storage_mode VARCHAR(32) NOT NULL DEFAULT 'json_table',
  enabled TINYINT(1) NOT NULL DEFAULT 1,
  sort_no INT NOT NULL DEFAULT 0,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_module_code (module_code)
);
```

内置模块：LOCATION（库位）、CONTAINER（容器）、PRODUCT（商品），storage_mode 统一为 `json_table`。

## RBAC 权限系统

五表标准 RBAC 模型，用户 → 角色 → 菜单权限。

```
sys_user ──多对多── sys_user_role ── sys_role
                                        │
                                   sys_role_menu
                                        │
                                     sys_menu (directory/page/button)
```

- 登录 `/auth/login` 返回 token + 权限码列表 + 菜单树
- 前端路由守卫 + v-permission 指令控制页面和按钮
- roles/menus CRUD 完整接口

## 字段字典定义

字段存储在 `print_field_dict`，按 `module_code` 分组。运行时由 `schemaService.compileSchema(moduleCode)` 动态读取，无需硬编码映射。

`print_field_dict` 表中 `searchable` 列控制该字段是否在业务数据中支持精确查询（1=启用，0=不启用）。业务编码字段（主键）默认 `searchable=1` 且不可取消。

### 库位（LOCATION）

| 字段编码 | 名称 | 必填 | 可搜索 | 可排序 |
|---------|------|------|--------|--------|
| locationCode | 库位编码 | 是 | 是 | 是 |
| locationPrefix | 库位前缀 | — | 是 | 是 |
| row | 排 | — | — | — |
| column | 列 | — | — | — |
| level | 层 | — | — | — |
| directionMark | 方向标 | — | — | — |
| warehouseCode | 区域仓编码 | 是 | 是 | 是 |
| areaCode | 物理仓编码 | — | 是 | 是 |

### 容器（CONTAINER）

| 字段编码 | 名称 | 必填 | 可搜索 | 可排序 |
|---------|------|------|--------|--------|
| containerCode | 容器编码 | 是 | 是 | 是 |
| warehouseCode | 区域仓编码 | — | 是 | 是 |
| areaCode | 物理仓编码 | — | 是 | 是 |

### 商品（PRODUCT）

| 字段编码 | 名称 | 必填 | 可搜索 | 可排序 |
|---------|------|------|--------|--------|
| productCode | 商品编码 | 是 | 是 | 是 |
| ProductBarcode | 商品条码 | — | 是 | 是 |
| customerProductCode | 客户商品编码 | — | 是 | 是 |

### 方向标规则

```
direction_flag = 1  → 显示"向上" → 打印 ↑
direction_flag = 2  → 显示"向下" → 打印 ↓
direction_flag = 空 → 显示空 → 打印空
```

## 存储说明

所有业务数据字段以 JSON 存储在 `business_record.record_data` 中。搜索时：
- **业务编码搜索**：按 `record_code` 列匹配（支持逗号/空格分隔批量查询）
- **字段精确查询**：按 `JSON_EXTRACT(record_data, '$."字段编码"')` 精确匹配，仅对启用了查询（`searchable=1`）的字段生效

`search_text` 列为历史可搜索字段值的拼接，当前搜索逻辑不再使用该列。后端由 `schemaService` 运行时编译模块和字段 schema，`recordRepository` 统一执行 CRUD。
