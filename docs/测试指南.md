# 测试指南

本文档说明项目的测试文件结构与运行方式。

## 后端测试 (`backend/tests/`)

| 文件 | 测试范围 |
|------|----------|
| `api.test.js` | API 接口集成测试 |
| `dsl.test.js` | DSL 模板校验逻辑 |
| `dslParity.test.js` | DSL 前后端一致性 |
| `env.test.js` | 环境变量加载 |
| `pdfLayout.test.js` | PDF 布局生成 |
| `recordService.test.js` | 业务记录 CRUD 服务 |

### 运行测试

```bash
cd backend

# 运行全部测试
npm test

# 运行数据库相关测试（需配置 .env.test）
RUN_DB_TESTS=1 npm run test:db
```

---

## 前端测试 (`frontend/tests/`)

| 文件 | 测试范围 |
|------|----------|
| `authFetchService.test.js` | API 请求鉴权封装 |
| `barcodeHumanTextService.test.js` | 条码可读文本生成 |
| `downloadService.test.js` | 文件下载逻辑 |
| `resizeService.test.js` | 元素缩放计算 |
| `templateDesignerLoader.test.js` | 模板设计器加载流程 |
| `validationService.test.js` | 表单校验规则 |

### 运行测试

```bash
cd frontend
npx vitest run
```
