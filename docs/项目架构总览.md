## 核心设计原则

### 1. 统一业务数据存储
所有业务数据（库位/容器/商品/自定义模块）统一存储在 `business_record` 表中，字段schema由 `print_field_dict` 运行时动态编译，**无需硬编码映射**。

### 2. 运行时字段编译
`schemaService.compileSchema(moduleCode)` 在运行时读取 `print_business_module` + `print_field_dict`，动态生成表单配置、校验规则和搜索字段，新增模块无需修改后端代码。

### 3. RBAC 权限控制
- 前端：路由守卫（router.beforeEach）+ `v-permission` 指令控制页面和按钮
- 后端：`requirePermission` 中间件验证每个API调用的权限码
- 权限变更需用户重新登录生效

### 4. 模板设计器
可视化拖拽设计，元素通过 `print_template_element` JSON 存储。打印时由 `pdfLayout.js` 解析元素配置，结合 `pdfGenerator.js` + `bwip-js` + `qrcode` 生成 PDF。
