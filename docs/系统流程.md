# 系统流程

本文档以通俗易懂的方式说明打印模板中心核心业务流程，帮助业务人员理解系统如何工作。

---

## 1. 登录流程

### 1.1 账号登录

```
1. 打开系统登录页面
2. 输入分配给您的账号和密码
3. 点击「登录」按钮
```

**系统内部做了什么**：
- 验证账号密码是否正确
- 验证通过后，系统生成一个"通行证"（JWT token）
- 系统返回您的个人信息、可以使用的功能列表、左侧菜单结构
- 浏览器把这些信息保存起来，后续操作凭借这个通行证自动识别身份

### 1.2 登录失败的情况

| 情况 | 原因 |
|------|------|
| 账号或密码错误 | 输入的账号或密码不正确 |
| 账号已停用 | 该账号已被管理员停用 |
| 长时间未操作 | 通行证过期，需要重新登录 |

## 2. 新建并启用模板流程

### 2.1 整体流程概览

```
创建空白模板 → 设计模板内容 → 保存模板 → 启用模板 → 开始使用
```

### 2.2 创建空白模板

1. 进入「模板列表」页面
2. 点击「新增模板」按钮
3. 填写模板基本信息（名称、类型、尺寸）
4. 点击「确定」

**此时系统做了什么**：
- 在数据库中创建了一条模板记录，状态为"停用"

### 2.3 设计模板内容

1. 自动进入模板设计器界面
2. 从左侧拖拽需要的组件到中间画布
3. 点击组件，在右侧设置属性和绑定字段
4. 重复以上步骤直到设计完成

### 2.4 保存模板

1. 点击「保存」按钮
2. 系统检查模板内容是否正确：
   - **有问题**：显示错误提示，修改后重新保存
   - **没问题**：保存成功

**保存时系统检查什么**：
- 模板名称是否填写了
- 尺寸是否有效（宽高都要大于0）
- 画布上是否有添加组件
- 动态文字/二维码/条码是否绑定了具体字段
- 组件是否放在画布范围内

### 2.5 启用模板

1. 返回模板列表页面
2. 找到刚保存的模板
3. 点击操作列的「启用」按钮
4. 系统再次检查模板内容
5. 启用成功，状态变为绿色"可用"

**为什么要启用步骤**：
- 保存只是把内容存起来，启用才表示"这个模板可以正式用了"
- 启用前再做一次检查，确保模板可以正常打印

### 2.6 启用失败怎么办

常见问题和解决方法：

| 问题 | 解决方法 |
|------|----------|
| 画布上没有组件 | 先拖拽组件到画布上 |
| 动态组件没绑定字段 | 点击组件，在右侧绑定字段 |
| 组件完全超出画布 | 把组件拖回画布范围内 |
| 二维码/条码尺寸太小 | 调大二维码/条码的宽高（至少12mm） |

## 3. 打印流程

### 3.1 整体流程概览

```
准备数据 → 选择模板 → 设置份数 → 生成PDF → 下载打印
```

### 3.2 详细步骤

1. **进入业务数据页面**
   - 路径：左侧菜单「业务数据」

2. **选择业务类型**
   - 在顶部 Tab 选择：库位 / 容器 / 商品

3. **勾选要打印的数据**
   - 点击数据左侧的复选框，可选一条或多条

4. **点击「打印」按钮**

5. **选择模板**
   - 左侧显示可用的模板列表
   - 点击选择要使用的模板
   - 右侧预览区域显示模板效果

6. **设置打印份数**
   - 输入要打印的份数（1-99份）

7. **点击「确认打印」**

8. **系统生成 PDF 文件并自动下载**

### 3.3 系统内部做了什么

```
收到打印请求
       │
       ▼
检查模板是否存在且已启用
       │
       ▼
检查业务数据类型与模板类型是否匹配
       │
       ▼
遍历每一条要打印的数据
       │
       ├── 静态文字 → 直接显示
       ├── 动态文字 → 替换成实际数据值
       ├── 二维码 → 生成二维码图片
       └── 条码 → 生成条码图片
       │
       ▼
组合所有内容生成 PDF 文件
       │
       ▼
记录打印日志（谁、什么时间、打印了什么）
       │
       ▼
返回 PDF 文件，浏览器下载
```

### 3.4 打印失败怎么办

| 问题 | 解决方法 |
|------|----------|
| 找不到模板 | 检查模板列表，确保模板状态是"启用" |
| 类型不匹配 | 库位数据要用库位模板，商品数据要用商品模板 |
| 二维码/条码显示不出来 | 检查是否绑定了字段，尺寸是否足够大 |

## 4. 业务数据管理流程

### 4.1 新增数据

1. 进入「业务数据」页面
2. 切换到目标业务类型 Tab
3. 点击「新增」按钮
4. 在表单中填写各项数据
5. 点击「确定」

**注意事项**：
- 带 * 号的字段是必填的，必须填写
- 主键字段（如库位编码）不能和已有数据重复

### 4.2 修改数据

1. 在数据列表中，点击要修改的那一行的编码列
2. 在弹出表单中修改数据内容
3. 点击「确定」

**注意**：主键字段（如库位编码）一般不能修改

### 4.3 批量导入数据

当需要一次性导入大量数据时，用 Excel 导入更高效：

```
1. 点击「导入」按钮
2. 点击「下载导入模板」下载 Excel 文件
3. 在 Excel 中按照格式要求填写数据
4. 上传填写好的 Excel 文件
5. 点击「确认导入」
6. 查看导入结果
   - 成功：数据已全部导入
   - 有失败：查看失败原因，修改后重新导入
```

**导入模板说明**：
- 第一行是表头，不能修改
- 从第二行开始填写数据
- 主键字段（如库位编码）不能留空或重复
- 日期字段有特定格式要求

### 4.4 批量删除数据

1. 勾选要删除的多条数据（可利用搜索先筛选）
2. 点击「删除」按钮
3. 在确认框中点击「确定」

### 4.5 多字段查询

业务数据页面支持按多个字段组合查询：

1. 在"业务编码"输入框中可输入编码（支持逗号或空格分隔批量查询）
2. 搜索区域会根据模版字段中启用了"启用查询"的字段，自动显示对应的输入框
3. 在需要筛选的字段输入框中输入值
4. 点击「查询」按钮，系统按所有填写的条件精确匹配
5. 点击「重置」清空所有查询条件

**说明**：哪些字段出现在搜索区域，取决于模版字段页面中该字段是否勾选了"启用查询"。业务编码字段默认启用查询且不可取消。

## 5. 字段管理流程

### 5.1 理解字段和模块的关系

- **模块**：一组相关字段的分类，例如"库位"、"商品"
- **字段**：模块下的具体数据项，例如"库位编码"、"商品条码"

### 5.2 新建业务模块

1. 进入「模版字段」页面
2. 点击「新增模块」按钮
3. 填写模块信息
4. 点击「确定」

**新建模块时会自动做什么**：
- 创建模块记录
- 自动创建一个主键字段（例如库位模块会创建"库位编码"字段）

### 5.3 新增字段

1. 切换到目标模块的 Tab 页
2. 点击「新增字段」按钮
3. 填写字段信息（包括是否勾选"启用查询"）
4. 点击「确定」

**"启用查询"说明**：
- 勾选后，该字段在业务数据页面会显示对应的搜索框，支持按该字段精确查询
- 默认勾选；业务编码字段（主键）强制勾选且不可取消
- 不勾选的字段不会出现在业务数据的搜索区域

### 5.4 停用和启用字段

- **停用**：字段暂时不使用，但保留在系统中（已有数据不受影响）
- **启用**：将停用的字段重新恢复使用

**适用场景**：
- 某个字段暂时不需要了，但以后可能还会用到
- 字段对应的业务流程暂停，暂时不收集这个数据

### 5.5 删除字段

1. 点击操作列的「删除」按钮
2. 在确认框中点击「确定」

**注意**：
- 必填字段不允许删除，需先取消必填或停用该字段
- 删除字段不会影响已有的业务数据记录
- 只是在模板设计器中不能再使用这个字段了

## 6. 权限管理流程

### 6.1 理解权限体系

系统采用"角色+权限"的方式控制用户可以操作哪些功能：

```
用户 → 角色 → 功能权限
```

- **用户**：使用系统的人
- **角色**：一组权限的集合，例如"仓库主管"、"录入员"
- **功能权限**：具体可以操作哪些菜单和按钮

### 6.2 管理员如何给用户分配权限

1. 进入「系统管理」→「角色管理」
2. 点击要编辑的角色操作列的「编辑」
3. 在权限卡片中：
   - 勾选页面 = 给这个角色开放该页面的访问权限
   - 点击蓝色的权限标签 = 给这个角色开放该按钮的操作权限
4. 点击「确定」保存

### 6.3 权限生效的时机

- 用户重新登录后，新分配的权限才会生效
- 如果用户已经登录，需要退出重新登录

### 6.4 常见权限场景

| 场景 | 需要的权限 |
|------|-----------|
| 查看模板列表 | template:view |
| 新建和编辑模板 | template:create + template:edit |
| 启用/停用模板 | template:enable + template:disable |
| 管理业务数据 | business:view + business:create + business:edit + business:delete |
| 导入数据 | business:import |
| 打印数据 | business:print |
| 管理用户 | system:user:* |
| 管理角色和菜单 | system:role:* + system:menu:* |

### 7.2 各部分数据说明

| 数据类型 | 存储内容 | 使用场景 |
