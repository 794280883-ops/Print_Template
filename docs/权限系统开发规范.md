# 权限系统开发规范

## 架构

```
用户 ──多对多── 角色 ──多对多── 菜单(权限码)
```

5 张表: `sys_user`, `sys_role`, `sys_menu`, `sys_user_role`, `sys_role_menu`

菜单 `type` 分三类:
- `directory` — 侧边栏分组，不可勾选
- `page` — 页面，可独立勾选(控制可见性)
- `button` — 按钮/操作，依赖页面

## 新增页面

### 1. 添加菜单记录

在 `backend/migrations/002_rbac_tables.sql` 的菜单种子中追加:

```sql
(50, 0, '新模块', 'directory', NULL, NULL, NULL, 40, 1),
(51, 50, '新页面', 'page', '/new-page', 'module:view', 'IconOutlined', 10, 1),
```

- `parent_id` = 目录 ID 或 0(顶级)
- `permission_code` 格式: `模块:操作`
- 注意避开已有 ID（当前最大 43）

### 2. 更新 admin 角色范围

```sql
-- 修改末尾行
SELECT 1, id FROM sys_menu WHERE id BETWEEN 1 AND 51;
```

### 3. 前端路由

```js
// router/index.js
{
  path: '/new-page',
  meta: { permission: 'module:view', title: '新页面', icon: 'IconOutlined', menuGroup: '新模块' },
  component: () => import('../views/NewPage.vue'),
}
```

## 新增按钮

### 1. 添加菜单记录

```sql
(52, 51, '新增xxx', 'button', NULL, 'module:create', NULL, 10, 1),
(53, 51, '删除xxx', 'button', NULL, 'module:delete', NULL, 20, 1),
```

- `parent_id` = 所属页面的菜单 ID
- `permission_code` 格式: `模块:操作`

### 2. 前端按钮

```html
<!-- 页面级按钮 -->
<a-button type="primary" @click="handleAdd" v-permission="'module:create'">新增</a-button>

<!-- 行操作按钮(普通元素) -->
<a-button size="small" v-permission="'module:delete'" @click="handleDelete(row)">删除</a-button>

<!-- 行操作按钮(弹窗确认) — 必须用 v-if，不能用 v-permission -->
<a-popconfirm v-if="hasPermission('module:delete')" @confirm="handleDelete(row)">
  <a-button size="small" danger>删除</a-button>
</a-popconfirm>
```

**重要**: `<a-popconfirm>` 和 `<a-tooltip>` 等 Ant Design 包裹组件不能使用 `v-permission` 指令，必须用 `v-if`:

```html
<!-- ❌ 错误 -->
<a-popconfirm v-permission="'xxx:delete'">...</a-popconfirm>

<!-- ✅ 正确 -->
<a-popconfirm v-if="hasPermission('xxx:delete')">...</a-popconfirm>
```

### 3. 需要 `hasPermission` 的页面

在 `<script setup>` 中导入:

```js
import { usePermissionStore } from '../../stores/permission.js';
const { hasPermission } = usePermissionStore();
```

## 权限码命名规范

```
模版:   template:view / create / edit / delete / enable / disable / preview
字段:   field:view / create / edit / delete / enable / disable
        field:module:create / module:edit / module:delete
业务:   business:view / create / edit / delete / import / print
系统:   system:user:view / create / edit / delete / disable / password
        system:role:view / create / edit / delete
        system:menu:view / create / edit / delete
自定义: 模块名:操作名
```

## 角色权限分配

1. 角色管理 → 编辑角色
2. 卡片式界面, 每组一个卡片
3. 勾选页面 checkbox = 授予查看权限 + 显示按钮配置区
4. 点击蓝色 tag = 授予/撤销按钮权限
5. 「全选」= 勾选页面 + 全部按钮
6. 「清空」= 取消所有

## 测试清单

新增页面/按钮后, 必须验证:

- [ ] admin 可访问
- [ ] 仅授予查看权限时, 页面可见但按钮不可操作
- [ ] 无权限时, 直接访问 URL → 跳转 `/403`
- [ ] 无权限的菜单目录不显示在侧边栏
- [ ] 编码/名称列: 有 edit 权限时显示链接, 无权限时显示纯文本

## 常见错误

1. **v-permission 在 popconfirm 上不生效** → 改用 `v-if`
2. **修改菜单名称后侧边栏仍是旧名** → 需重新登录
3. **docker exec 传中文 SQL 乱码** → 编辑 `.sql` 文件, 用 `npm run migrate`
4. **角色权限变更后不生效** → 需要重新登录刷新 session
